# 国际化修改总结：消除硬编码英文

## 修改概述

将所有用户可见的硬编码英文文本提取到 `strings.xml` 资源文件，支持多语言（英文 + 简体中文）。

---

## 文件变更清单

### 新增文件

| 文件 | 说明 |
|------|------|
| `app/src/main/res/values/strings.xml` | 英文字符串资源（默认） |
| `app/src/main/res/values-zh-rCN/strings.xml` | 中文字符串资源（简体中文） |

### 修改文件

| 文件 | 修改内容 |
|------|---------|
| `NetTestScreen.kt` | 全部硬编码文本 → `stringResource(R.string.xxx)` |
| `EvaluationScreen.kt` | 全部硬编码文本 → `stringResource(R.string.xxx)` |
| `MainActivity.kt` | Toast 消息 → `getString(R.string.xxx)` |
| `NetTestViewModel.kt` | 错误消息 → `context.getString(R.string.xxx)` |

## 代码修改示例

### 1. NetTestScreen.kt - 按钮文本

**修改前：**
```kotlin
Text("Start Test", fontSize = 14.sp)
```

**修改后：**
```kotlin
Text(stringResource(R.string.start_test), fontSize = 14.sp)
```

### 2. NetTestScreen.kt - 对话框标题

**修改前：**


**修改前：**
```kotlin
Toast.makeText(this, "Permissions granted", Toast.LENGTH_SHORT).show()
```

## 添加新语言的步骤

如需支持其他语言（如日语、韩语等）：

1. **创建新的 values 目录**：
```bash
app/src/main/res/values-ja/    # 日语
app/src/main/res/values-ko/    # 韩语
```

2. **复制并翻译 strings.xml**：
```bash
cp app/src/main/res/values/strings.xml app/src/main/res/values-ja/strings.xml
# 然后编辑翻译内容
```

3. **Android 会自动根据系统语言选择对应的资源文件**

## 注意事项

1. **字符串格式化**：使用 `stringResource(R.string.key, arg1, arg2)` 传递参数
2. **复数形式**：如需复数，使用 `<plurals>` 资源（本项目暂无）
3. **上下文相关翻译**：某些词在不同上下文可能需要不同翻译，本项目通过不同键名区分
4. **动态文本**：如时间、数字等动态内容，使用 `String.format` 或 `StringTemplate` 配合资源字符串

## 下一步（可选）

如需实现**应用内语言切换**（不依赖系统语言），需要：

1. 创建 `LocaleHelper.kt` 管理语言配置
2. 在 `Application` 或 `MainActivity` 中重写 `attachBaseContext` 应用语言
3. 提供语言选择 UI（如对话框或设置页）
4. 重启 Activity 使语言生效

但这已超出当前修改范围。当前方案已支持通过**系统语言设置**自动切换英文/中文。
