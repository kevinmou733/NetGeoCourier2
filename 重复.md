# NetTestScreen 与 NetTestViewModel 冲突分析及修改方案

## 冲突说明

`NetTestScreen.kt` 和 `NetTestViewModel.kt` 存在严重的逻辑重复和架构冲突：

### 1. 业务逻辑重复

**NetTestScreen.kt (第87-142行)** 独立实现了：
- `doTest()` - 执行网络测试
- `startAutoTest()` - 开始自动测试
- `stopAutoTest()` - 停止自动测试

**NetTestViewModel.kt (第61-117行)** 也实现了相同的逻辑。

### 2. 状态管理不一致

| 状态 | Screen 实现 | ViewModel 实现 | 问题 |
|------|------------|---------------|------|
| 测试中 | `var isTesting by mutableStateOf(false)` | `_isTesting StateFlow` | 两套状态不同步 |
| 自动测试 | `var isAutoTesting by mutableStateOf(false)` | `_isAutoTesting StateFlow` | 数据无法共享 |
| 测试结果 | `var testResults by mutableStateOf(listOf<...>)` | `_testResults StateFlow` | 数据冗余 |
| 当前结果 | `var currentResult by mutableStateOf<...>` | `_curResult StateFlow` | 状态不一致 |

### 3. 职责混淆（违反 MVVM）

**NetTestScreen.kt 直接调用**：
- `SpeedTestHelper.measureDownloadSpeed()` (第99行)
- `SpeedTestHelper.measureUploadSpeed()` (第100行)
- `SpeedTestHelper.measurePing()` (第101行)
- `FileHelper.saveCsv()` (第216行)
- `FileHelper.saveAmapHtml()` (第246行)
- `FileHelper.sendEmail()` (第255行)

**NetTestViewModel.kt 已封装**：
- 所有上述 helper 调用都在 ViewModel 中通过函数暴露

### 4. MainActivity 混合使用

MainActivity.kt (第117行) 从 ViewModel 读取 `currentPage` 状态，但 NetTestScreen 维护独立状态，导致旋转屏幕时数据丢失。

---

## 修改方案：Screen 只负责 UI，全部逻辑交回 ViewModel

### 修改 1：NetTestScreen.kt - 删除所有状态定义（第77-84行）

**原代码：**
```kotlin
77:     var isTesting by remember { mutableStateOf(false) }
78:     var isAutoTesting by remember { mutableStateOf(false) }
79:     var autoJob by remember { mutableStateOf<Job?>(null) }
80:     var testResults by remember { mutableStateOf(listOf<NetTestResult>()) }
81:     var currentResult by remember { mutableStateOf<NetTestResult?>(null) }
82:     var csvPath by remember { mutableStateOf<String?>(null) }
83:     var htmlPath by remember { mutableStateOf<String?>(null) }
84:     val coroutineScope = rememberCoroutineScope()
85:     val scrollState = rememberScrollState()
```

**修改为：**
```kotlin
77:     val viewModel: NetTestViewModel = viewModel()
78:     val isTesting by viewModel.isTesting.collectAsState()
79:     val isAutoTesting by viewModel.isAutoTesting.collectAsState()
80:     val testResults by viewModel.testResults.collectAsState()
81:     val currentResult by viewModel.curResult.collectAsState()
82:     val csvPath by viewModel.csvPath.collectAsState()
83:     val htmlPath by viewModel.htmlPath.collectAsState()
84:     val coroutineScope = rememberCoroutineScope()
85:     val scrollState = rememberScrollState()
```

### 修改 2：NetTestScreen.kt - 删除 doTest 函数（第87-118行）

**完全删除：**
```kotlin
87:     fun doTest(onFinish: (() -> Unit)? = null) {
88:         isTesting = true
89:         currentResult = null
90:         coroutineScope.launch {
91:             val location = locationHelper.getCurrentLocation()
92:             if (location == null) {
93:                 Toast.makeText(context, "Failed to get location.", Toast.LENGTH_SHORT).show()
94:                 isTesting = false
95:                 onFinish?.invoke()
96:                 return@launch
97:             }
98: 
99:             val download = SpeedTestHelper.measureDownloadSpeed()
100:             val upload = SpeedTestHelper.measureUploadSpeed()
101:             val ping = SpeedTestHelper.measurePing()
102: 
103:             val timeText = SimpleDateFormat("yyyy-MM-dd HH:mm:ss", Locale.getDefault()).format(Date())
104:             val result = NetTestResult(
105:                 timestamp = timeText,
106:                 latitude = location.latitude,
107:                 longitude = location.longitude,
108:                 upload = upload,
109:                 download = download,
110:                 ping = ping
111:             )
112: 
113:             currentResult = result
114:             testResults = testResults + result
115:             isTesting = false
116:             onFinish?.invoke()
117:         }
118:     }
```

### 修改 3：NetTestScreen.kt - 删除 startAutoTest 函数（第120-135行）

**完全删除：**
```kotlin

121:         if (isAutoTesting) {
122:             return
123:         }

129:                 deferred.await()

132:                 }

```

### 修改 4：NetTestScreen.kt - 删除 stopAutoTest 函数（第137-142行）

**完全删除：**
```kotlin

138:         isAutoTesting = false
139:         autoJob?.cancel()
140:         autoJob = null
141:         isTesting = false
142:     }
```

### 修改 5：NetTestScreen.kt - 修改按钮点击调用（第188-262行）

将所有函数调用改为 viewModel 方法：

**第188行（Start Test 按钮）：**
```kotlin
// 原代码：
188:                             onClick = { doTest() },

// 修改为：
188:                             onClick = { viewModel.doTest() },
```

**第197行（Stop 按钮）：**
```kotlin
// 原代码：
197:                             onClick = { stopAutoTest() },

// 修改为：
197:                             onClick = { viewModel.stopAutoTest() },
```

**第226行（Auto Test 按钮）：**
```kotlin
// 原代码：
226:                             onClick = {
227:                                 if (isAutoTesting) stopAutoTest() else startAutoTest()
228:                             },

// 修改为：
226:                             onClick = {
227:                                 if (isAutoTesting) viewModel.stopAutoTest() else viewModel.startAutoTest()
228:                             },
```

**第216行（Save CSV 按钮）：**
```kotlin
// 原代码：
216:                             onClick = { csvPath = FileHelper.saveCsv(context, testResults) },

```

**第246行（Generate Map 按钮）：**
```kotlin
// 原代码：
246:                             onClick = { htmlPath = FileHelper.saveAmapHtml(context, testResults) },

// 修改为：

```

**第255行（Send Email 按钮）：**
```kotlin

```

### 修改 6：NetTestViewModel.kt - 添加缺失的依赖

需要在 ViewModel 中添加 `viewModel()` 可组合函数的导入：

**第26行后添加：**
```kotlin
import androidx.compose.runtime.saveable.rememberSaveable
import androidx.lifecycle.viewmodel.compose.viewModel
```

(注：如果项目中已配置 `androidx.lifecycle:lifecycle-viewmodel-compose` 依赖，则可直接使用)

### 修改 7：MainActivity.kt - 传递 ViewModel 给 NetTestScreen（可选）

**当前代码（第118-122行）：**
```kotlin
118:                         AppPage.TEST -> NetTestScreen(
119:                             locationHelper = locationHelper,
120: 
121:                             onOpenEvaluation = { viewmodel.currentPage = AppPage.EVALUATION }
122:                         )
```

**建议改为（传递 viewModel）：**
```kotlin
118:                         AppPage.TEST -> NetTestScreen(
119:                             viewModel = viewmodel,
120:                             onOpenEvaluation = { viewmodel.currentPage = AppPage.EVALUATION }
121:                         )
```

同时修改 NetTestScreen 函数签名：
```kotlin
// 原函数签名（第72-75行）：
72: @Composable
73: fun NetTestScreen(
74:     locationHelper: LocationHelper,
75:     onOpenEvaluation: () -> Unit
76: ) {

// 修改为：
72: @Composable
73: fun NetTestScreen(
74:     viewModel: NetTestViewModel,
75:     onOpenEvaluation: () -> Unit
76: ) {
```

**或者不传递，在 Screen 内部创建**（当前修改方案已采用）：
```kotlin
val viewModel: NetTestViewModel = viewModel()
```

## 简化效果

### 代码减少：
- NetTestScreen.kt 删除约 70 行（第77-142行 + 部分按钮逻辑）
- 逻辑更清晰：Screen 只负责 UI，ViewModel 负责业务

### 状态统一：
- 所有状态由 ViewModel 管理
- 旋转屏幕数据不丢失
- Compose 自动 recomposition

### 符合 MVVM：
- View 层（Screen）不包含业务逻辑
- ViewModel 层统一处理数据和操作
- 易于测试和维护
