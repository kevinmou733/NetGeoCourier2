## 📋 一、项目概述

### 1.1 项目简介

**NetGeoCourier2** 是一个基于 Android 的网络测速与定位应用，主要功能包括：

- 网络速度测试（下载/上传/Ping）
- GPS 定位（使用高德地图 SDK）
- 数据导出（CSV/HTML 地图）
- 网络评估报告（对接后端 API）

### 1.2 技术栈

## 🔴 二、报错原因分析

### 2.1 编译失败（阻塞性问题）

**错误信息：**
```
FAILURE: Build failed with an exception.

* What went wrong:
25.0.2
```

**错误位置：** `compile_errors.txt`, `compile_output.txt`

#### 根本原因：Java 版本不兼容

| 项目 | 配置值 | 实际值 | 状态 |
|------|--------|--------|------|
| **项目要求的 Java 版本** | 11 (LTS) | - | ⚠️ 未满足 |
| **系统当前 Java 版本** | - | 25.0.2 (2026) | ✗ 过高 |
| **AGP 支持的最高版本** | JDK 21 | - | ✗ Java 25 不支持 |
| **Gradle 版本** | 8.11.1 | 8.11.1 | ✓ 匹配 |
| **AGP 版本** | 8.10.1 | 8.10.1 | ✓ 匹配 |

**详细说明：**

1. **项目配置要求 Java 11**
   - `app/build.gradle.kts:52-59` 明确设置：
     ```kotlin
     compileOptions {
         sourceCompatibility = JavaVersion.VERSION_11
         targetCompatibility = JavaVersion.VERSION_11
     }
     kotlinOptions {
         jvmTarget = "11"
     }
     ```

2. **系统安装的是 Java 25.0.2**
   - 2026 年发布的最新 LTS 版本
   - **AGP 8.10.1 不支持 Java 25**（只支持到 JDK 21）
   - 导致构建时版本验证失败

3. **错误机制**
   - Gradle wrapper 调用系统的 `java` 命令
   - AGP 8.10.1 检查 Java 版本
   - 发现 `25.0.2` 超出支持范围
   - 抛出异常，仅显示版本号

#### ✅ 修复方案

**方案一：降级 Java 版本（推荐）**

1. 下载安装 **JDK 11** 或 **JDK 17**（LTS）
   - 推荐：Eclipse Adoptium Temurin
   - 下载地址：https://adoptium.net/

2. 配置 `JAVA_HOME` 环境变量：
   ```cmd
   setx JAVA_HOME "C:\Program Files\Eclipse Adoptium\jdk-17.0.12+7"
   setx PATH "%JAVA_HOME%\bin;%PATH%"
   ```

3. 验证版本：
   ```cmd
   java -version
   # 期望输出：openjdk version "17.0.x" 或 "11.x.x"
   ```

4. 清理并重新构建：
   ```cmd
   cd D:\NetGeoCourier2
   gradlew clean
   gradlew assembleDebug
   ```

**方案二：升级 AGP 版本（不推荐，风险高）**

修改 `gradle/libs.versions.toml`：
```toml
agp = "8.13.0"  # 或更高版本支持 Java 25
```

⚠️ 风险：可能引发 Compose、Kotlin 等依赖不兼容问题。

**方案三：指定 JDK 路径（临时）**

在 `gradle.properties` 添加：
```properties
org.gradle.java.home=C:\\Program Files\\Eclipse Adoptium\\jdk-11.0.24+8
```

### 2.2 源代码中的严重问题（运行时错误）

#### 🔴 问题 1：高德 SDK 方法名拼写错误

**影响文件：** `LocationHelper.kt:142`  
**严重级别：** 🔴 Critical（会导致运行时崩溃）

**错误代码：**
```kotlin
locationClient?.unRegisterLocationListener(this)  // 大写 R ❌
```

**正确写法：**
```kotlin
locationClient?.unregisterLocationListener(this)  // 全小写 ✅
```

**影响：**
- 编译可能通过（如果 SDK 使用动态绑定）
- 运行时可能抛出 `NoSuchMethodError`
- 定位监听器无法正确移除 → 资源泄漏
- 后续定位请求可能冲突

**证据：**
高德官方文档和示例代码均使用 `unregisterLocationListener`（全小写）。

---

#### 🔴 问题 2：双重坐标转换（Double Coordinate Conversion）

**影响范围：**
- `FileHelper.kt:35`
- `NetTestScreen.kt:352-353`
- `NetTestScreen.kt:413-414`

**问题描述：**

高德定位 SDK 默认返回 **GCJ-02（火星坐标系）**坐标，但代码错误地对其进行了第二次 WGS-84 → GCJ-02 转换。

**数据流：**
```
1. LocationHelper.getCurrentLocation() 获取 AMapLocation
2. AMapLocation.latitude/longitude → 已经是 GCJ-02 坐标
3. 存储到 NetTestResult（lines 73-74）
4. FileHelper.saveCsv() 调用 CoordTransform.wgs84ToGcj02() (line 35)
   - 输入：已经是 GCJ-02 的坐标
   - 输出：错误的坐标（双重转换）⚠️
5. NetTestScreen 显示坐标时也调用了转换 (lines 353, 414)
   - UI 显示同样错误
```

**后果：**
- 导出的 CSV 文件坐标偏差可达 **数百米**
- 地图显示位置严重偏离真实位置
- 网络测试结果失去地理参考价值

**修复方案：**

高德 SDK 返回的是 GCJ-02，直接使用即可，**不需要再转换**。

```kotlin
// ❌ 错误：双重转换
val (gcjLat, gcjLon) = CoordTransform.wgs84ToGcj02(it.latitude, it.longitude)

// ✅ 正确：直接使用
val gcjLat = it.latitude
val gcjLon = it.longitude
```

#### 🟡 问题 3：重复初始化防护缺失

**影响文件：** `LocationHelper.kt`  
**严重级别：** 🟡 High

**问题：**

多次调用 `getCurrentLocation()` 会创建新的 `AMapLocationClient` 实例，但旧的实例可能未正确销毁。

**当前代码：**
```kotlin
suspend fun getCurrentLocation() = suspendCancellableCoroutine { cont ->
    // 每次都新建客户端，未检查是否已有实例
    locationClient = AMapLocationClient(context.applicationContext).apply {
        setLocationListener(this@LocationHelper)
        ...
        startLocation()
    }
}
```

**风险：**
- 多个定位客户端同时运行 → 资源冲突
- 电池消耗激增
- 定位回调混乱

**建议修复：**
```kotlin
private var locationClient: AMapLocationClient? = null

suspend fun getCurrentLocation() = suspendCancellableCoroutine { cont ->
    // 如果已有客户端在运行，先停止
    locationClient?.stopLocation()
    locationClient?.unregisterLocationListener(this@LocationHelper)

    locationClient = AMapLocationClient(context.applicationContext).apply {
        setLocationListener(this@LocationHelper)
        ...
        startLocation()
    }
}
```

#### 🟢 问题 4：错误信息丢失

**影响文件：** `LocationHelper.kt:124-127`  
**严重级别：** 🟢 Medium

**问题：**

定位失败时只打印日志，未向上层传递具体错误信息。

```kotlin
it.errorCode.let { errorCode ->
    android.util.Log.e("LocationHelper", "定位失败, 错误码: $errorCode, 错误信息: ${it.errorInfo}")
}
locationContinuation?.resume(null)  // ❌ 无法区分失败原因
```

**建议：** 返回一个 `Result<AMapLocation>` 或自定义错误类型。

```kotlin
suspend fun getCurrentLocation() = suspendCancellableCoroutine { cont ->
    // 先停止之前的定位客户端
    locationClient?.stopLocation()
    locationClient?.unregisterLocationListener(this)

    if (!PermissionHelper.hasLocationPermission(context)) {
        cont.resume(null)
        return@suspendCancellableCoroutine
    }

    locationContinuation = cont

    try {
        locationClient = AMapLocationClient(context.applicationContext).apply {
            setLocationListener(this@LocationHelper)
            setLocationOption(createLocationOption())
            startLocation()
        }
    } catch (e: Exception) {
        e.printStackTrace()
        cont.resume(null)
    }

    cont.invokeOnCancellation {
        stopLocation()
        locationContinuation = null
    }
}
```

### ✅ 第三步：增强错误处理（可选）

修改 `LocationHelper.kt` 的 `onLocationChanged`，返回具体错误：

```kotlin
sealed class LocationResult {
    data class Success(val location: AMapLocation) : LocationResult()
    data class Failure(val errorCode: Int, val errorInfo: String) : LocationResult()
    object PermissionDenied : LocationResult()
    object Unknown : LocationResult()
}

override fun onLocationChanged(location: AMapLocation?) {
    stopLocation()

    when {
        location == null -> locationContinuation?.resume(LocationResult.Unknown)
        it.errorCode == 0 -> locationContinuation?.resume(LocationResult.Success(location))
        else -> locationContinuation?.resume(
            LocationResult.Failure(it.errorCode, it.errorInfo ?: "Unknown error")
        )
    }

    locationContinuation = null
}
```
