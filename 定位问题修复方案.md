# 定位失败问题诊断与修复方案

## 问题现象

权限已授予，但点击 **Start Test** 按钮后显示：
```
Failed to get location. Please check permissions.
```

但实际上权限已经授予，问题不在权限，而在**高德定位初始化**或**定位服务本身**。

---

## 根因分析

### 1. **高德定位 API Key 未正确初始化**（主要问题）

LocationHelper 中没有调用 `AMapLocationClient.setApiKey(apiKey)`，虽然 Manifest 中有 meta-data，但高德 SDK **某些版本需要代码显式调用**。

### 2. **缺少详细错误传递**

LocationHelper 的 `getCurrentLocation()` 返回 `null`，不区分：
- 权限被拒
- API Key 无效
- GPS 关闭
- 网络错误
- 定位超时

用户看到的是通用错误，无法得知真实原因。

### 3. **权限检查使用错误的 Context**

LocationHelper 构造时传入的 `context` 如果不是 Application Context，可能导致权限检查不准确。

---

## 修复方案

### 方案 A：快速修复（推荐）- 添加 API Key 初始化 + 详细错误日志

#### 步骤 1：LocationHelper.kt - 添加 API Key 初始化

在 `init` 块中读取 Manifest 的 API Key 并调用 `AMapLocationClient.setApiKey()`：

```kotlin
class LocationHelper(private val context: Context) : AMapLocationListener {
    
    private var locationClient: AMapLocationClient? = null
    private var locationContinuation: kotlin.coroutines.Continuation<AMapLocation?>? = null

    init {
        try {
            // 从 Manifest 读取 API Key
            val apiKey = context.packageManager
                .getApplicationInfo(context.packageName, 128)
                .metaData
                ?.getString("com.amap.api.v2.apikey")
            
            if (!apiKey.isNullOrEmpty()) {
                AMapLocationClient.setApiKey(apiKey)
                Log.d(TAG, "AMap API Key set: ${apiKey.take(5)}...")
            } else {
                Log.w(TAG, "AMap API Key not found in manifest")
            }
        } catch (e: Exception) {
            Log.e(TAG, "Failed to set AMap API Key", e)
        }
    }

    // ... 其余代码
}
```

#### 步骤 2：LocationHelper.kt - 增强日志

在 `onLocationChanged` 中添加详细日志：
```kotlin
override fun onLocationChanged(location: AMapLocation?) {
    Log.d(TAG, "onLocationChanged called with location: $location")
    
    stopLocation()

    location?.let {
        Log.d(TAG, "Location errorCode: ${it.errorCode}, errorInfo: ${it.errorInfo}")
        if (it.errorCode == 0) {
            Log.d(TAG, "Location success: lat=${it.latitude}, lon=${it.longitude}")
            locationContinuation?.resume(it)
        } else {
            Log.e(TAG, "Location failed: errorCode=${it.errorCode}, errorInfo=${it.errorInfo}")
            locationContinuation?.resume(null)
        }
    } ?: run {
        Log.w(TAG, "Location is null")
        locationContinuation?.resume(null)
    }

    locationContinuation = null
}
```

#### 步骤 3：NetTestViewModel.kt - 添加错误日志

```kotlin
fun doTest(onFinish: (() -> Unit)? = null, onError: (String) -> Unit = {}) {
    // ...
    coroutineScope.launch {

            _isTesting.value = false
            onError(context.getString(R.string.failed_get_location))
            onFinish?.invoke()

        }
        // ...
```

### 方案 B：完善错误处理 - 返回具体错误码（更优）

#### 步骤 1：定义 LocationResult 密封类

在 LocationHelper.kt 中添加：
```kotlin
sealed class LocationResult {
    data class Success(val location: AMapLocation): LocationResult()
    data class Failure(val errorCode: Int, val errorInfo: String): LocationResult()
}
```

#### 步骤 2：修改 getCurrentLocation 返回 LocationResult

```kotlin
suspend fun getCurrentLocation(): LocationResult = suspendCancellableCoroutine { cont ->
    if (!PermissionHelper.hasLocationPermission(context)) {
        cont.resume(LocationResult.Failure(-1, "No location permission"))
        return@suspendCancellableCoroutine
    }

    locationContinuation = cont

    try {
        locationClient = AMapLocationClient(context.applicationContext).apply {
            setLocationListener(this@LocationHelper)
            setLocationOption(createLocationOption())
            startLocation()
        }
    } catch (e: Exception) {
        Log.e(TAG, "Failed to start location client", e)
        cont.resume(LocationResult.Failure(-2, "Client init failed: ${e.message}"))
    }

    cont.invokeOnCancellation {
        stopLocation()
        locationContinuation = null
    }
}
```

修改 `onLocationChanged`：
```kotlin
override fun onLocationChanged(location: AMapLocation?) {
    stopLocation()

    location?.let { loc ->
        if (loc.errorCode == 0) {
            Log.d(TAG, "Location success")
            locationContinuation?.resume(LocationResult.Success(loc))
        } else {
            Log.e(TAG, "Location failed: errorCode=${loc.errorCode}, errorInfo=${loc.errorInfo}")
            locationContinuation?.resume(LocationResult.Failure(loc.errorCode, loc.errorInfo ?: "Unknown"))
        }
    } ?: run {
        Log.w(TAG, "Location is null")
        locationContinuation?.resume(LocationResult.Failure(-3, "Location callback null"))
    }

    locationContinuation = null
}
```

#### 步骤 3：修改 NetTestViewModel.kt 处理 LocationResult

```kotlin
fun doTest(onFinish: (() -> Unit)? = null, onError: (String) -> Unit = {}) {
    if (_isTesting.value || _isAutoTesting.value) return
    
    _isTesting.value = true
    _curResult.value = null
    
    coroutineScope.launch {
        val context = getApplication<Application>()
        when (val result = locationHelper.getCurrentLocation()) {
            is LocationResult.Success -> {
                val location = result.location
                val download = SpeedTestHelper.measureDownloadSpeed()
                val upload = SpeedTestHelper.measureUploadSpeed()
                val ping = SpeedTestHelper.measurePing()

                val timeStr = SimpleDateFormat("yyyy-MM-dd HH:mm:ss", Locale.getDefault()).format(Date())
                val netResult = NetTestResult(
                    timestamp = timeStr,
                    latitude = location.latitude,
                    longitude = location.longitude,
                    upload = upload,
                    download = download,
                    ping = ping
                )

                _curResult.value = netResult
                _testResults.value = _testResults.value + netResult
                _isTesting.value = false
                onFinish?.invoke()
            }
            is LocationResult.Failure -> {
                _isTesting.value = false
                val errorMessage = when (result.errorCode) {
                    -1 -> "Location permission denied. Please grant location permission."
                    -2 -> "Location client initialization failed: ${result.errorInfo}"
                    -3 -> "Location returned null"
                    12 -> "Missing required location permission (manifest or runtime)"
                    4 -> "Network error. Check internet connection."
                    5 -> "GPS is disabled. Please enable location services."
                    6 -> "Device network closed or not available"
                    15 -> "AMap API Key is invalid. Check your API key configuration."
                    else -> "Location failed (${result.errorCode}): ${result.errorInfo}"
                }
                onError(errorMessage)
                onFinish?.invoke()
            }
        }
    }
}
```

### 方案 C：最终简化版（立即使用）

如果觉得方案 B 改动太大，采用方案 A 并添加一个简单的**位置服务检查**：

在 ViewModel 的 doTest 开始前检查：
```kotlin
val lm = context.getSystemService(Application.LOCATION_SERVICE) as LocationManager
val gpsEnabled = lm.isProviderEnabled(LocationManager.GPS_PROVIDER)
val networkEnabled = lm.isProviderEnabled(LocationManager.NETWORK_PROVIDER)
if (!gpsEnabled && !networkEnabled) {
    onError("Location services are disabled. Please enable GPS or network location.")
    return
}
```

## 推荐实施顺序

1. **先实施方案 A**（API Key 初始化 + 详细日志）
   - 修改 LocationHelper.kt（添加 init 块和日志）
   - 重新编译运行
   - 查看 Logcat 中 `LocationHelper` 标签的日志
   - 确定失败的错误码

2. **根据错误码决定下一步**
   - 如果是错误码 15 (API Key 无效) → 检查 highmap 平台配置
   - 如果是错误码 5 (GPS 关闭) → 提示用户开启位置服务
   - 如果是错误码 4 (网络错误) → 检查网络
   - 其他 → 根据错误信息处理

3. **如果方案 A 仍无法解决**，升级到方案 B，返回具体错误码给用户

## 快速验证步骤

### 1. 查看 Logcat
```bash
adb logcat | grep LocationHelper
```

### 2. 检查 Manifest 中 API Key 是否已替换

构建后检查 `app/build/intermediates/merged_manifest/debug/AndroidManifest.xml`：
```bash
grep -A1 "com.amap.api.v2.apikey" app/build/intermediates/merged_manifest/debug/AndroidManifest.xml
```

确认 value 不是 `${AMAP_KEY}`，而是实际 Key。

### 3. 验证高德 API Key 有效性

登录 [高德开放平台](https://lbs.amap.com/)：
- 检查 Android Key 是否已启用
- 确认包名：`com.example.netgeocourier`
- 确认 SHA1：调试版 SHA1 用 `./gradlew signingReport` 获取

### 4. 测试定位服务

确保测试设备：
- ✅ 位置服务已开启（GPS 或 网络定位）
- ✅ 已授予应用定位权限
- ✅ 网络连接正常

---

## 常见高德定位错误码

| 错误码 | 含义 | 解决方案 |
|--------|------|---------|
| 0 | 成功 | - |
| 1 | 无效的定位参数 | 检查定位参数配置 |
| 2 | 定位服务中断 | 重启定位 |
| 4 | 网络连接失败 | 检查网络 |
| 5 | 定位失败，GPS 关闭 | 引导用户开启 GPS |
| 6 | 设备网络关闭 | 开启网络 |
| 12 | 定位失败，缺少权限 | 检查 Manifest 和运行时权限 |
| 15 | API Key 无效 | 检查 Key 配置 |
| 16 | 未初始化 API Key | 调用 `setApiKey()` |
