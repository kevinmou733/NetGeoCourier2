## 🎯 第一部分：项目是做什么的？

### 一句话概括

**NetGeoCourier2** 是一个 **"网络测速 + 定位 + 报告生成"** 的 Android 手机应用。

### 具体功能

想象一下，你是一个**快递员**或者**网络检测员**，你需要：

1. **📍 获取当前位置**
   - 打开应用，点击按钮
   - 应用获取你的 GPS 坐标（经度、纬度）
   - 显示在地图上

2. **🚀 测试网络速度**
   - 点击"开始测速"
   - 应用自动：
     - 下载一个大文件 → 测下载速度
     - 上传一个大文件 → 测上传速度
     - Ping 一个服务器 → 测延迟
   - 显示结果：**下载 85 Mbps / 上传 23 Mbps / 延迟 28ms**

3. **💾 保存测试记录**
   - 每次测速的结果自动保存
   - 可以查看历史记录
   - 以图表形式显示速度变化趋势

4. **📧 导出数据**
   - 导出为 **CSV 表格文件**（可以用 Excel 打开）
   - 生成 **HTML 地图网页**（点击直接在浏览器打开，显示所有测速点在地图上的位置）
   - 通过邮件发送给管理员

5. **📊 查看网络评估报告**
   - 连接后端服务器
   - 获取专业的网络质量评估报告
   - 查看评分、建议等

###### LocationHelper.kt（定位助手）

**作用**: 使用高德地图 SDK 获取当前位置

**工作原理**（类比叫外卖）：

```
1. 你(应用)说: "我要定位！"
   ↓
2. LocationHelper 检查: "你有GPS权限吗？"（PermissionHelper）
   ↓ 有权限？
   ├─ 否 → 返回失败
   ↓ 有权限
3. LocationHelper 说: "高德，帮我定位！"（创建 AMapLocationClient）
   ↓
4. 高德 SDK 开始工作（打开GPS）
   ↓
5. 高德找到位置后回调: "找到了！坐标是 xxx, yyy"（onLocationChanged）
   ↓
6. LocationHelper 把坐标返回给调用者
```

**关键代码解读**:

```kotlin
class LocationHelper(private val context: Context) : AMapLocationListener {
    // 高德定位客户端（就像"对讲机"）
    private var locationClient: AMapLocationClient? = null
    
    // 协程 continuation（就像"回call电话"）
    private var locationContinuation: Continuation<AMapLocation?>? = null
    
    // 获取定位的主函数
    suspend fun getCurrentLocation() = suspendCancellableCoroutine { cont ->
        // 1. 检查权限
        if (!PermissionHelper.hasLocationPermission(context)) {
            cont.resume(null)  // 没权限，直接返回空
            return@suspendCancellableCoroutine
        }
        
        // 2. 保存回调（等会儿高德找到了位置，好告诉调用者）
        locationContinuation = cont
        
        // 3. 创建高德客户端，开始定位
        locationClient = AMapLocationClient(context.applicationContext).apply {
            setLocationListener(this@LocationHelper)  // 设置回调监听
            setLocationOption(createLocationOption())  // 配置定位参数
            startLocation()  // 开始定位！
        }
    }
    
    // 高德定位成功或失败的回调函数
    override fun onLocationChanged(location: AMapLocation?) {
        stopLocation()  // 定位完了，关掉对讲机
        
        location?.let {
            if (it.errorCode == 0) {
                // ✅ 成功！把位置返回给调用者
                locationContinuation?.resume(it)
            } else {
                // ❌ 失败，也返回空
                locationContinuation?.resume(null)
            }
        }
        
        locationContinuation = null
    }
}
```

###### SpeedTestHelper.kt（测速工具）

**作用**: 测试网络速度

**测试原理**（类比水管放水）：

```
📥 下载速度测试:
1. 准备一个大水管（25MB 数据）
2. 连接到 Cloudflare 测速服务器
3. 开始计时 ⏱️
4. 让水（数据）流进水桶（手机）
5. 水桶满了（25MB接收完）→ 停止计时
6. 计算: 25MB × 8 = 200 Mb / 用时 = 速度 (Mbps)

📤 上传速度测试:
1. 准备 2MB 的水（数据）
2. 连接到 Cloudflare 服务器
3. 开始计时 ⏱️
4. 把水从水桶泼出去（上传）
5. 服务器确认收到 → 停止计时
6. 计算: 2MB × 8 = 16 Mb / 用时 = 速度 (Mbps)

🏓 Ping 测试:
1. 向 1.1.1.1（Cloudflare DNS）发送"ping"信号
2. 记录发送时间
3. 收到回复 → 计算往返时间
4. 结果: 28ms（表示延迟28毫秒）
```

**关键代码**:

```kotlin
object SpeedTestHelper {
    // 测速服务器地址
    private const val DOWNLOAD_URL = "https://speed.cloudflare.com/__down?bytes=25000000"
    private const val UPLOAD_URL = "https://speed.cloudflare.com/__up"
    private const val PING_HOST = "1.1.1.1"
    
    // 测下载速度
    suspend fun measureDownloadSpeed(): Double = withContext(Dispatchers.IO) {
        val connection = URL(DOWNLOAD_URL).openConnection() as HttpURLConnection
        val buffer = ByteArray(32 * 1024)  // 32KB 缓冲区
        var downloadedBytes = 0L
        val startTime = System.currentTimeMillis()
        
        // 循环读取数据直到读完
        connection.inputStream.use { input ->
            var len: Int
            while (input.read(buffer).also { len = it } != -1) {
                downloadedBytes += len
            }
        }
        
        val endTime = System.currentTimeMillis()
        val duration = (endTime - startTime) / 1000.0  // 转换为秒
        
        // 速度 = 字节数 × 8（转成比特） ÷ 时间 ÷ 1,000,000（转成Mbps）
        (downloadedBytes * 8) / (duration * 1_000_000)
    }
    
    // 测上传速度（原理类似）
    suspend fun measureUploadSpeed(): Double = withContext(Dispatchers.IO) {
        val data = ByteArray(2 * 1024 * 1024)  // 2MB 数据
        val connection = URL(UPLOAD_URL).openConnection() as HttpURLConnection
        connection.doOutput = true
        connection.requestMethod = "POST"
        
        val start = System.currentTimeMillis()
        connection.outputStream.use { it.write(data) }  // 写入数据（上传）
        val code = connection.responseCode
        val end = System.currentTimeMillis()
        
        val seconds = (end - start) / 1000.0
        (data.size * 8) / (seconds * 1_000_000)
    }
    
    // 测 Ping（延迟）
    suspend fun measurePing(): Int = withContext(Dispatchers.IO) {
        val address = InetAddress.getByName(PING_HOST)
        val startTime = System.currentTimeMillis()
        val isReachable = address.isReachable(3000)  // 3秒超时
        val duration = System.currentTimeMillis() - startTime
        if (isReachable) duration.toInt() else -1
    }
}
```

###### CoordTransform.kt（坐标翻译官）

**作用**: 在不同坐标系统之间转换

**为什么需要坐标转换？**

因为不同地图使用不同的坐标系统：

| 地图服务 | 坐标系统 | 说明 |
|----------|----------|------|
| GPS、Google Maps | WGS-84 | 国际标准，原始 GPS 坐标 |
| 高德地图、腾讯地图 | GCJ-02 | 中国标准，对 WGS-84 加偏 |
| 百度地图 | BD-09 | 在 GCJ-02 基础上二次加偏 |

**⚠️ 关键知识点**:

> **高德定位 SDK 返回的坐标已经是 GCJ-02！**
> 
> 所以**不需要**调用 `CoordTransform.wgs84ToGcj02()` 再次转换！
> 
> 再次转换会导致坐标偏差**数百米**（双重转换 Bug）

**坐标转换算法**（简化说明）：

```kotlin
object CoordTransform {
    // WGS-84 转 GCJ-02
    fun wgs84ToGcj02(lat: Double, lon: Double): Pair<Double, Double> {
        // 1. 检查是否在中国境内（否则不转换）
        if (outOfChina(lat, lon)) return lat to lon
        
        // 2. 应用复杂的数学变换（涉及三角函数）
        //    （这是国家测绘局的加密算法）
        
        // 3. 返回转换后的坐标
        return (lat + dLat) to (lon + dLon)
    }
}
```

###### AuthTokenStore.kt（保险箱）

**作用**: 安全地保存登录令牌（Token）

**工作原理**（类比酒店房卡）：

```
1. 用户登录成功
   ↓
2. 服务器返回 accessToken（就像一张"房卡"）
   ↓
3. AuthTokenStore 把房卡存到"保险箱"
   ↓
   SharedPreferences（Android 的轻量级存储）
   ↓
4. 以后每次请求 API，从保险箱取房卡
   ↓
5. 在请求头加上: Authorization: Bearer <token>
   ↓
6. 服务器验证房卡，允许访问
```

**关键代码**:

```kotlin
object AuthTokenStore {
    private const val PREFS_NAME = "auth_prefs"
    private const val KEY_ACCESS_TOKEN = "access_token"
    
    // 保存 Token
    fun saveAccessToken(context: Context, token: String) {
        context.getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE)
            .edit()
            .putString(KEY_ACCESS_TOKEN, normalizeToken(token))
            .apply()
    }
    
    // 读取 Token
    fun getAccessToken(context: Context): String? {
        return context.getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE)
            .getString(KEY_ACCESS_TOKEN, null)
            ?.let(::normalizeToken)  // 自动去除 "Bearer " 前缀
    }
    
    // 清除 Token（退出登录）
    fun clearAccessToken(context: Context) {
        context.getSharedPreferences(PREFS_NAME, Context.MODE_PRIVATE)
            .edit()
            .remove(KEY_ACCESS_TOKEN)
            .apply()
    }
    
    // 规范化：如果 token 是 "Bearer xxx"，提取 "xxx"
    private fun normalizeToken(rawToken: String): String {
        return if (rawToken.startsWith("Bearer ", ignoreCase = true)) {
            rawToken.substringAfter(' ').trim()
        } else {
            rawToken
        }
    }
}
```

**作用**: 配置 Retrofit（网络请求库），创建 API 服务实例

**类比**: 就像打电话前要配置电话线路

```kotlin
object ApiClient {
    // 单例模式：整个应用共用一个 Retrofit 实例
    @Volatile
    private var evaluationApiService: EvaluationApiService? = null
    
    fun evaluationService(context: Context): EvaluationApiService {
        // 双重检查锁：确保只创建一次
        return evaluationApiService ?: synchronized(this) {
            evaluationApiService ?: buildRetrofit(context.applicationContext)
                .create(EvaluationApiService::class.java)
                .also { evaluationApiService = it }
        }
    }
    
    private fun buildRetrofit(context: Context): Retrofit {
        // 1. 创建 OkHttpClient（网络底层库）
        val okHttpClient = OkHttpClient.Builder()
            .addInterceptor(AuthInterceptor(context))  // 添加认证拦截器
            .connectTimeout(15, TimeUnit.SECONDS)      // 连接超时15秒
            .readTimeout(15, TimeUnit.SECONDS)         // 读取超时15秒
            .writeTimeout(15, TimeUnit.SECONDS)        // 写入超时15秒
            .build()
        
        // 2. 创建 Retrofit 实例
        return Retrofit.Builder()
            .baseUrl(BuildConfig.API_BASE_URL)  // 基础 URL（从 buildConfig 读取）
            .client(okHttpClient)               // 使用配置好的 OkHttp
            .addConverterFactory(GsonConverterFactory.create())  // JSON 解析
            .build()
    }
}
```

###### AuthInterceptor.kt（安检员）

**作用**: 在每个网络请求发送前，自动添加 Authorization 头

**比喻**: 就像进公司要刷卡，安检员帮你刷卡

```kotlin
class AuthInterceptor(private val context: Context) : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        val originalRequest = chain.request()
        
        // 1. 从保险箱取 Token
        val token = AuthTokenStore.getAccessToken(context)
        
        // 2. 如果有 Token，添加到请求头
        val request = if (token.isNullOrBlank()) {
            originalRequest  // 没有 Token，原样发送
        } else {
            originalRequest.newBuilder()
                .header("Authorization", "Bearer $token")  // 添加认证头
                .build()
        }
        
        // 3. 继续发送请求
        return chain.proceed(request)
    }
}
```

###### EvaluationRepository.kt（数据采购员）

**作用**: 处理数据获取的业务逻辑，隔离 UI 和网络

**为什么需要 Repository？**

```
直接调用 (❌ 不推荐):
NetTestScreen → ApiClient → 网络请求
    ↑                     ↑
UI层              网络层（紧耦合）

使用 Repository (✅ 推荐):
NetTestScreen → Repository → ApiClient
    ↑                ↑             ↑
  UI层       业务逻辑层      网络层（松耦合）
```

**好处**:
1. UI 不直接依赖网络细节
2. 可以方便地切换数据源（比如从网络切换到本地缓存）
3. 便于测试（可以 mock Repository）

**关键代码**:

```kotlin
class EvaluationRepository(
    private val apiService: EvaluationApiService
) {
    suspend fun getEvaluation(): Result<EvaluationData> = withContext(Dispatchers.IO) {
        runCatching {
            // 1. 发送请求
            val response = apiService.getEvaluation()
            
            // 2. 检查 HTTP 状态码
            if (!response.isSuccessful) {
                throw IOException("Request failed: HTTP ${response.code()}")
            }
            
            // 3. 解析响应体
            val body = response.body() ?: throw IOException("Response body is empty.")
            
            // 4. 检查业务状态码（后端自定义的 success 字段）
            if (!body.success || body.data == null) {
                throw IOException(body.message.ifBlank { "Evaluation request failed." })
            }
            
            // 5. 返回数据
            body.data
        }
    }
}
```

**返回类型 `Result<EvaluationData>`**:
- `Result.success(data)` - 成功
- `Result.failure(exception)` - 失败（包含异常信息）

```
      - 添加 AuthInterceptor（自动加 Authorization 头）
      - 发送 GET 请求到 http://10.0.2.2:3000/api/v1/evaluation
       ↓
    后端服务器 (Node.js):
      - 验证 JWT Token
      - 查询该用户的所有测速记录
      - 计算平均下载、平均 Ping、平均 RSSI、平均 SNR
      - 计算综合评分 (score) 和等级 (level)
      - 生成优化建议 (suggestions)
      - 返回 JSON 响应
       ↓
    Repository 接收响应:
      - 检查 response.isSuccessful
      - 检查 body.success == true
      - 提取 body.data
      - 返回 Result.success(EvaluationData)
       ↓
    EvaluationScreen 更新 UI:
      - uiState = EvaluationUiState(data = result)
      - 显示: EvaluationSummaryCard (分数、等级)
      - 显示: MetricsCard (各项指标)
      - 显示: SuggestionsCard (建议列表)
  }
```

## 🎓 第六部分：核心概念通俗解释

### 1. 什么是 Activity（活动）？

**比喻**: Activity 就像电影的**一个场景**或**一页幻灯片**

```
SplashActivity  →  启动页场景（显示 Logo，1 秒后切换）
MainActivity    →  主场景（包含两个页面: 测速页、评估页）
```

**特点**:
- 每个 Activity 是一个独立的屏幕
- 用户可以切换 Activity（就像翻页）
- 本项目只用了 2 个 Activity（符合"单 Activity 架构"趋势）

---

### 2. 什么是 Compose？

**比喻**: Compose 就像**搭积木**，而不是**画油画**

**传统方式（XML 布局）**:
```
1. 先在 XML 文件画好布局（像画油画）
2. 然后在 Java/Kotlin 代码里 findViewById 找控件
3. 修改控件属性
```

**Compose 方式（声明式 UI）**:
```kotlin
@Composable
fun NetTestScreen(...) {
    Column {
        Text("标题")
        Button(onClick = { ... }) {
            Text("按钮")
        }
    }
}
```

**优势**:
- 代码即 UI，不用 XML
- 状态驱动：数据变了 UI 自动更新
- 更简洁、更易维护

---

### 3. 什么是 ViewModel？

**比喻**: ViewModel 就像**客厅的智能中控屏**

```
手机旋转屏幕（Configuration Change）
    ↓
Activity 会重建（所有变量丢失）
    ↓
但 ViewModel 保留（像中控屏记住所有状态）
    ↓
新 Activity 从 ViewModel 读取数据
    ↓
UI 恢复原状
```

**作用**:
1. 保存 UI 状态（即使屏幕旋转也不丢）
2. 处理业务逻辑（与 UI 解耦）
3. 生命周期感知（自动清理资源）

---

### 4. 什么是 Repository（仓库）？

**比喻**: Repository 就像**超市的仓库管理员**

```
UI (顾客) → Repository (管理员) → 网络/数据库 (仓库)

顾客不需要知道货从哪来，只找管理员要
管理员负责从仓库取货
```

**好处**:
- UI 层不直接调用网络
- 可以轻松替换数据源（比如从网络切换到本地缓存）
- 便于测试（可以 mock 管理员）

---

### 5. 什么是协程（Coroutine）？

**比喻**: 协程就像**轻量级的线程**，但不用手动管理

**传统多线程问题**:
```
线程1: 网络请求（等待 5 秒）
线程2: UI 更新（等待线程1完成）
线程3: 数据库写入（等待网络请求）
→ 线程太多，管理复杂
```

**协程解决方案**:
```kotlin
// suspend 函数（可暂停的函数）
suspend fun getLocation() {
    delay(5000)  // 等待 5 秒（不阻塞线程！）
    return location
}

// 调用
lifecycleScope.launch {
    val loc = getLocation()  // 挂起，不卡 UI
    updateUI(loc)           // 恢复执行
}
```

**关键点**:
- `suspend` = 可暂停（像函数中间可以"暂停"，稍后继续）
- `launch` = 启动一个协程
- 自动线程切换（IO 操作在后台，UI 操作在主线程）

---

### 6. 什么是 Jetpack Compose State（状态）？

**比喻**: State 就像**智能粘合剂**

```
var count by remember { mutableStateOf(0) }

Button(onClick = { count++ }) {
    Text("Count: $count")
}
```

**工作原理**:
1. `count` 是 State（可观察的数据）
2. `Text` 读取 `count`
3. 点击按钮 → `count++`
4. State 变化 → Compose 自动重新执行 `Text` 的代码
5. UI 更新

**关键词**:
- `mutableStateOf` - 创建可变状态
- `remember` - 记住状态（屏幕旋转不丢失）
- `by` - 委托（简化读写）
- 自动重组（Recomposition） - State 变化时自动刷新 UI

---

### 7. 什么是 Retrofit + OkHttp？

**比喻**:
- **OkHttp**: 像**快递公司**，负责实际运输 HTTP 请求
- **Retrofit**: 像**智能代理**，把 Java 接口方法自动转换成 HTTP 请求

```kotlin
interface ApiService {
    @GET("users/{id}")
    suspend fun getUser(@Path("id") id: Int): User
}

// 调用:
val user = apiService.getUser(123)
// Retrofit 自动发送: GET /users/123
// 并解析 JSON 成 User 对象
```

---

### 8. 什么是 Gradle？

**比喻**: Gradle 就像**项目的总承包商**

**职责**:
1. 下载依赖库（Retrofit、高德 SDK 等）
2. 编译 Kotlin/Java 代码
3. 打包 APK（安装包）
4. 运行测试
5. 签名发布

**常用命令**:
```bash
./gradlew clean        # 清理编译产物
./gradlew assembleDebug  # 编译调试版 APK
./gradlew installDebug   # 安装到设备
./gradlew build         # 完整构建
```

## 🔍 第八部分：常见问题解答（FAQ）

### Q1: 为什么要分这么多包（helper, network, screen）？

**答**: 为了**职责分离**，便于维护。

想象一个公司：
- `helper/` - 后勤部（提供工具）
- `network/` - 商务部（对外联系）
- `screen/` - 销售部（直接面对客户）
- `viewmodel/` - 总经理（协调各部门）

如果所有代码混在一起，就像一个人既当客服又当会计又当仓库管理员，容易混乱。

---

### Q2: 为什么 ViewModel 要使用 StateFlow？

**答**: 为了**响应式数据更新**。

传统方式:
```kotlin
// UI 主动查询数据（低效）
button.setOnClickListener {
    val result = viewModel.getResult()  // 每次都要调用
    textView.text = result.toString()
}
```

StateFlow 方式:
```kotlin
// 数据变化时自动通知 UI
viewModel.testResults.collect { results ->
    // 数据一变，这里自动执行
    textView.text = results.toString()
}
```

---

### Q3: 为什么要把 Token 存在 AuthTokenStore？

**答**: 为了**安全性和持久化**。

- **SharedPreferences** 是 Android 提供的轻量级存储
- 适合存小量数据（如 Token、设置）
- 应用重启后数据还在
- 比存文件更安全（内部存储，其他应用无法访问）

---

### Q4: 为什么用单例（object）？

**答**: 为了**全局唯一，避免重复创建**。

`object ApiClient`:
- Retrofit 实例只需要一个（复用连接池）
- 整个应用共享同一个实例

`object SpeedTestHelper`:
- 工具类，不需要实例化
- 直接调用 `SpeedTestHelper.measureDownloadSpeed()`

---

### Q5: 为什么有那么多 `?.`（安全调用）？

**答**: 防止**空指针异常**（NullPointerException）。

```kotlin
locationClient?.stopLocation()
```

等价于:
```kotlin
if (locationClient != null) {
    locationClient.stopLocation()
}
// 如果为 null，不执行，不报错
```

Kotlin 的类型系统区分：
- `String` - 不能为 null
- `String?` - 可以为 null
- `String!!` - 断言不为 null（危险！）

### 项目核心功能

1. **定位**: 使用高德 SDK 获取 GPS 坐标
2. **测速**: 使用 Cloudflare 测速服务
3. **存储**: 使用 SharedPreferences 保存 Token
4. **导出**: CSV + HTML 地图 + 邮件
5. **评估**: 对接后端 API 获取专业报告

### 技术栈总览

```
前端 (Android):
├─ 语言: Kotlin 2.0.0
├─ UI 框架: Jetpack Compose (Material 3)
├─ 架构: MVVM + Repository
├─ 异步: Kotlin Coroutines + StateFlow
├─ 网络: Retrofit 2.11.0 + OkHttp 4.12.0
├─ 定位: 高德 Android 定位 SDK 6.2.0
└─ 最低支持: Android 8.0 (API 26)

后端 (Node.js):
└─ Express + ? (待探索)
```

### 关键要点

1. **单一职责**: 每个文件只做一件事
2. **分层清晰**: UI → ViewModel → Repository → Service
3. **响应式**: StateFlow + Compose 自动更新
4. **安全**: Token 加密存储、权限检查
5. **用户体验**: 加载状态、错误提示、历史记录
